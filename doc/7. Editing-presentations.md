# Редактирование презентаций

---

## 1. Общая концепция

Редактор презентаций — центральный модуль QuizSlides, позволяющий авторизованному пользователю (Презентатору) конструировать интерактивные презентации из набора слайдов и виджетов.

### Архитектура редактора

Редактор построен на следующей иерархии данных:

```
Presentation
 └── Slide (number, picture, status)
      └── Widget
           ├── WordCloud  (вопрос + лимит слов)
           ├── Quiz       (таймер + возможность смены ответа)
           └── AnswerOption (варианты ответа)
```

| Сущность | Модель | Ключевые поля |
| :--- | :--- | :--- |
| **Презентация** | `Presentation` | `id_presentation`, `name`, `status`, `id_user` |
| **Слайд** | `Slide` | `id_slide`, `number`, `picture`, `status`, `id_presentation` |
| **Виджет** | `Widget` | `id_widget`, `id_slide` |
| **Облако слов** | `WordCloud` | `id_word_cloud`, `question`, `word_limit`, `id_widget` |
| **Опрос** | `Quiz` | `id_quiz`, `max_time`, `changeability`, `id_widget` |
| **Вариант ответа** | `AnswerOption` | `id_answer_option`, `number`, `text`, `id_widget` |

### Принципы работы

- **Автосохранение** — каждое действие (добавление, редактирование, удаление) немедленно отправляется на сервер через AJAX/Fetch-запрос; отдельной кнопки «Сохранить».
- **Авторизация** — все маршруты редактора защищены декоратором `@login_required`; пользователь может редактировать только свои презентации.
- **JSON API** — сервер принимает и возвращает данные в формате JSON (`JsonResponse`), фронтенд работает без перезагрузки страницы.
- **Порядок слайдов** — определяется полем `Slide.number`; при изменении порядка сервер пересчитывает значения `number` для всех затронутых слайдов.

## 2. Сценарии работы

### 2.1 Добавление виджета на слайд

| Элемент | Описание |
| :--- | :--- |
| **Актор** | Презентатор |
| **Предусловие** | Презентация создана, в ней существует хотя бы один слайд, пользователь авторизован. |
| **Основной сценарий** | 1. Презентатор выбирает целевой слайд в боковой панели редактора.<br>2. Презентатор нажимает кнопку **«Добавить виджет»** и выбирает тип: *Облако слов*, *Опрос* или *Варианты ответа*.<br>3. Фронтенд отправляет `POST /editor/<id_presentation>/slides/<id_slide>/widgets/` с указанием типа виджета и начальных параметров.<br>4. Сервер создаёт запись `Widget`, а также дочернюю запись (`WordCloud`, `Quiz` или `AnswerOption`) со значениями по умолчанию.<br>5. Сервер возвращает JSON с `id_widget` и данными созданного виджета.<br>6. Фронтенд отображает новый виджет на слайде и открывает панель его настроек. |
| **Альт. сценарий** | **Невалидные данные** (шаг 3) — сервер возвращает `400 Bad Request` с описанием ошибки; виджет не создаётся. |
| **Конечное состояние** | В БД сохранён новый виджет, привязанный к слайду; на экране отображается визуальное представление виджета. |

### 2.2 Редактирование виджета

| Элемент | Описание |
| :--- | :--- |
| **Актор** | Презентатор |
| **Предусловие** | На слайде существует хотя бы один виджет. |
| **Основной сценарий** | 1. Презентатор кликает на виджет на слайде.<br>2. Система открывает панель параметров, соответствующую типу виджета:<br>&nbsp;&nbsp;— *Облако слов*: поле вопроса, лимит слов.<br>&nbsp;&nbsp;— *Опрос*: время на ответ (`max_time`), возможность смены ответа (`changeability`).<br>&nbsp;&nbsp;— *Варианты ответа*: текст и порядковый номер каждого варианта.<br>3. Презентатор изменяет нужные параметры.<br>4. Фронтенд отправляет `PATCH /editor/<id_presentation>/slides/<id_slide>/widgets/<id_widget>/` с изменёнными полями.<br>5. Сервер валидирует данные и обновляет запись.<br>6. Сервер возвращает `200 OK` с обновлёнными данными виджета.<br>7. Фронтенд обновляет отображение виджета на слайде. |
| **Альт. сценарий** | **Виджет не найден** (шаг 4) — сервер возвращает `404 Not Found`. Фронтенд показывает уведомление об ошибке. |
| **Конечное состояние** | Изменения зафиксированы в БД; визуальное представление виджета обновлено. |

### 2.3 Удаление виджета

| Элемент | Описание |
| :--- | :--- |
| **Актор** | Презентатор |
| **Предусловие** | На слайде существует виджет, который требуется удалить. |
| **Основной сценарий** | 1. Презентатор наводит курсор на виджет и нажимает иконку **«Удалить»** (или использует контекстное меню).<br>2. Система показывает модальное окно подтверждения: *«Удалить виджет? Это действие необратимо»*.<br>3. Презентатор подтверждает удаление.<br>4. Фронтенд отправляет `DELETE /editor/<id_presentation>/slides/<id_slide>/widgets/<id_widget>/`.<br>5. Сервер удаляет запись `Widget` и все связанные дочерние записи (`WordCloud`, `Quiz`, `AnswerOption`) каскадно (`on_delete=CASCADE`).<br>6. Сервер возвращает `200 OK` с подтверждением.<br>7. Фронтенд убирает виджет со слайда. |
| **Альт. сценарий** | **Отмена удаления** (шаг 3) — Презентатор нажимает «Отмена»; запрос не отправляется, виджет остаётся на месте. |
| **Конечное состояние** | Виджет и его дочерние данные удалены из БД; слайд отображается без удалённого виджета. |

### 2.4 Изменение порядка слайдов

| Элемент | Описание |
| :--- | :--- |
| **Актор** | Презентатор |
| **Предусловие** | В презентации существует два или более слайдов. |
| **Основной сценарий** | 1. Презентатор в боковой панели захватывает миниатюру слайда и перетаскивает её (drag-and-drop) на новую позицию.<br>2. Фронтенд формирует новый массив порядка: `[{id_slide: 5, number: 1}, {id_slide: 2, number: 2}, ...]`.<br>3. Фронтенд отправляет `PATCH /editor/<id_presentation>/slides/reorder/` с массивом нового порядка.<br>4. Сервер в одной транзакции обновляет поле `number` у каждого слайда согласно полученному массиву.<br>5. Сервер возвращает `200 OK` с обновлённым списком слайдов.<br>6. Фронтенд обновляет боковую панель, отображая слайды в новом порядке. |
| **Альт. сценарий** | **Неполный массив** (шаг 3) — массив не содержит все слайды презентации; сервер возвращает `400 Bad Request` с ошибкой *«Переданы не все слайды»*. |
| **Конечное состояние** | Поле `number` каждого слайда обновлено; боковая панель отображает слайды в новом порядке. |

## 3. API маршруты редактора

Все маршруты расположены под префиксом `/editor/` и защищены `@login_required`. Формат обмена данными — JSON.

### Презентация

| Метод | URL | Описание | Тело запроса | Ответ |
| :--- | :--- | :--- | :--- | :--- |
| `POST` | `/create/` | Создание презентации | `title` — название | `{success, id, name}` |
| `GET` | `/editor/<id_presentation>/` | Получение презентации со всеми слайдами и виджетами | — | `{id_presentation, name, status, slides: [...]}` |
| `PATCH` | `/editor/<id_presentation>/` | Обновление метаданных презентации (имя, статус) | `{name?, status?}` | `{success, ...updated_fields}` |
| `DELETE` | `/editor/<id_presentation>/` | Удаление презентации и всех вложенных данных | — | `{success}` |

### Слайды

| Метод | URL | Описание | Тело запроса | Ответ |
| :--- | :--- | :--- | :--- | :--- |
| `GET` | `/editor/<id_presentation>/slides/` | Список слайдов презентации | — | `[{id_slide, number, picture, status}, ...]` |
| `POST` | `/editor/<id_presentation>/slides/` | Добавление нового слайда | `{picture?}` | `{id_slide, number}` |
| `PATCH` | `/editor/<id_presentation>/slides/<id_slide>/` | Обновление слайда (картинка, статус) | `{picture?, status?}` | `{success, ...updated_fields}` |
| `DELETE` | `/editor/<id_presentation>/slides/<id_slide>/` | Удаление слайда (с пересчётом `number`) | — | `{success}` |
| `PATCH` | `/editor/<id_presentation>/slides/reorder/` | Изменение порядка слайдов | `{order: [{id_slide, number}, ...]}` | `{success, slides: [...]}` |

### Виджеты

| Метод | URL | Описание | Тело запроса | Ответ |
| :--- | :--- | :--- | :--- | :--- |
| `GET` | `/editor/<id_presentation>/slides/<id_slide>/widgets/` | Список виджетов слайда | — | `[{id_widget, type, data}, ...]` |
| `POST` | `/editor/<id_presentation>/slides/<id_slide>/widgets/` | Создание виджета | `{type, ...params}` | `{id_widget, type, data}` |
| `PATCH` | `/editor/<id_presentation>/slides/<id_slide>/widgets/<id_widget>/` | Обновление виджета | `{...changed_fields}` | `{success, ...updated_fields}` |
| `DELETE` | `/editor/<id_presentation>/slides/<id_slide>/widgets/<id_widget>/` | Удаление виджета (каскадно) | — | `{success}` |


### Коды ответов

| Код | Значение |
| :--- | :--- |
| `200` | Успешная операция |
| `400` | Невалидные данные (отсутствуют обязательные поля, некорректные значения) |
| `403` | Пользователь не является владельцем презентации |
| `404` | Презентация, слайд или виджет не найдены |
| `405` | HTTP-метод не разрешён для данного маршрута |
