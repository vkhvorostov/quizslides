# Документация по вариантам использования системы QuizSlides

## Введение
QuizSlides – веб-платформа для создания и проведения интерактивных презентаций с опросами. Авторы регистрируются, создают слайды (контент или опрос), запускают публичные сессии. Зрители подключаются по коду, вводят имя и участвуют в опросах в реальном времени. Администратор управляет пользователями, модерацией, поддержкой, серверным ПО и базой данных.

## Акторы
- **Администратор** – полный контроль над системой и пользователями.
- **Автор (Авторизованный пользователь)** – создаёт презентации, управляет ими, проводит сессии.
- **Гость** – неавторизованный пользователь, подключается к активной сессии для просмотра и ответов на опросы.
- **Система** – внутренний актор, отвечающий за аутентификацию, хранение, синхронизацию и т.д.

## Диаграмма вариантов использования
- **Администратор**: Блокировка пользователей, Модерация, Техподдержка, Обновление ПО сервера, Доступ к БД и информации о пользователях.
- **Автор**: Создать аккаунт, Войти в аккаунт, Создать презентацию, Запустить презентацию, Переключить слайд.
- **Гость**: Присоединиться к презентации (включает «Ввод имени»).

## Спецификации вариантов использования

### UC-1. Войти в аккаунт

| Элемент | Описание |
|---------|----------|
| Наименование | Войти в аккаунт |
| Участники | Автор (будущий) → цель: получить доступ к функциям создания и управления конференциями.<br>Система → цель: аутентифицировать пользователя и выдать сессию с правами автора. |
| Предусловие | 1. Пользователь находится на странице входа.<br>2. У пользователя уже есть аккаунт (логин + пароль). |
| Результат успеха | Пользователь аутентифицирован, получает роль «Автор», открывается основной интерфейс автора (список конференций, кнопка создания новой и т.д.). |
| Триггер | Нажатие кнопки «Войти» после ввода учётных данных. |
| Основной сценарий | 1. Система получает логин и пароль.<br>2. Система находит пользователя по логину.<br>3. Система проверяет соответствие пароля.<br>4. Пароль верный.<br>5. Система создаёт сессию / токен.<br>6. Перенаправление в интерфейс автора.<br>7. Отображается имя пользователя и доступные функции. |
| Расширения | 2a. Логин не найден → сообщение «Пользователь не найден» + ссылка на регистрацию.<br>4a. Неверный пароль → сообщение «Неверный пароль» + счётчик попыток; при превышении – временная блокировка + ссылка «Восстановить пароль».<br>4b. Аккаунт заблокирован → сообщение о блокировке + предложение восстановить доступ. |

### UC-2. Создать аккаунт

| Элемент | Описание |
|---------|----------|
| Наименование | Создать аккаунт |
| Участники | Посетитель / будущий Автор → цель: зарегистрироваться для доступа к созданию конференций.<br>Система → цель: создать пользователя, сохранить данные безопасно и выдать начальный доступ. |
| Предусловие | 1. Пользователь находится на странице регистрации.<br>2. Пользователь не авторизован. |
| Результат успеха | Создан новый пользователь с ролью «Автор», пользователь сразу авторизован и перенаправлен в интерфейс автора. |
| Триггер | Нажатие кнопки «Зарегистрироваться» после заполнения формы. |
| Основной сценарий | 1. Система получает логин, пароль, (опционально имя).<br>2. Проверяет, что логин свободен.<br>3. Проверяет требования к паролю.<br>4. Создаёт запись в базе (хэш пароля).<br>5. Создаёт сессию.<br>6. Перенаправляет в интерфейс автора.<br>7. Показывает приветственный экран. |
| Расширения | 2a. Логин уже занят → «Этот логин уже используется» + кнопка «Войти».<br>4a. Пароль слабый → список ошибок, возврат к форме. |

### UC-3. Создать презентацию

| Элемент | Описание |
|---------|----------|
| Наименование | Создать презентацию |
| Участники | Автор → цель: создать новую презентацию со слайдами.<br>Система → цель: сохранить структуру презентации в базе данных. |
| Предусловие | Автор авторизован, находится на главной странице интерфейса автора. |
| Результат успеха | Создана новая презентация с заданным названием и слайдами, отображается в списке презентаций автора. |
| Триггер | Нажатие кнопки «Новая презентация». |
| Основной сценарий | 1. Автор нажимает «Новая презентация».<br>2. Система запрашивает название презентации.<br>3. Автор вводит название и подтверждает создание.<br>4. Система предлагает добавить слайды: «Контент» или «Опрос».<br>5. При выборе «Контент» автор может добавить текст, картинку, диаграмму.<br>6. При выборе «Опрос» автор вводит название опроса, варианты ответов, опционально правильный ответ и время ответа.<br>7. Автор может изменить порядок слайдов (перетаскивание).<br>8. Автор может удалить слайд.<br>9. Система сохраняет презентацию и отображает её в библиотеке. |
| Расширения | 4a. Автор не добавляет ни одного слайда – презентация сохраняется как пустая (только название).<br>3a. Автор отменяет создание – система удаляет черновик и возвращает к списку. |

### UC-4. Запустить презентацию

| Элемент | Описание |
|---------|----------|
| Наименование | Запустить презентацию |
| Участники | Автор → цель: начать публичный показ презентации.<br>Система → цель: сгенерировать сессию и код доступа, перейти в режим демонстрации. |
| Предусловие | Автор авторизован, выбрана готовая презентация из списка. |
| Результат успеха | Сгенерирован уникальный код сессии, отображается в интерфейсе автора; слайды переведены в режим показа; зрители могут подключаться по коду. |
| Триггер | Нажатие кнопки «Старт» на странице презентации. |
| Основной сценарий | 1. Автор открывает нужную презентацию и нажимает «Старт».<br>2. Система генерирует уникальный идентификатор сессии и отображает его (например, слева) для подключения зрителей.<br>3. Система переходит в режим демонстрации экрана (отображается первый слайд).<br>4. В процессе показа автор может переключать слайды (см. UC-5). |
| Расширения | 2a. В презентации нет слайдов → сообщение «Добавьте хотя бы один слайд перед запуском».<br>3a. Автор закрывает сессию – система завершает показ, зрители отключаются. |

### UC-5. Переключить слайд

| Элемент | Описание |
|---------|----------|
| Наименование | Переключить слайд |
| Участники | Автор → цель: перемещаться по слайдам во время показа.<br>Система → цель: обновить отображаемый слайд у автора и всех подключённых зрителей. |
| Предусловие | Презентация запущена (активна сессия). |
| Результат успеха | На экране автора и всех зрителей отображается предыдущий или следующий слайд. |
| Триггер | Нажатие клавиш навигации (стрелки вправо/влево) или клик по области слайда. |
| Основной сценарий | 1. Автор нажимает клавишу стрелки или кликает по слайду.<br>2. Система определяет направление переключения (вперёд/назад) и индекс целевого слайда.<br>3. Система загружает соответствующий слайд.<br>4. Система синхронизирует отображение нового слайда со всеми подключёнными зрителями (через WebSocket). |
| Расширения | 2a. Переключение вперёд на последнем слайде → ничего не происходит или отображается сообщение «Презентация завершена».<br>2b. Переключение назад на первом слайде → ничего не происходит. |

### UC-6. Присоединиться к презентации

| Элемент | Описание |
|---------|----------|
| Наименование | Присоединиться к презентации |
| Участники | Гость → цель: смотреть презентацию и участвовать в интерактивах.<br>Система → цель: проверить код сессии, идентифицировать гостя и синхронизировать слайды. |
| Предусловие | Гость перешёл по ссылке или находится на странице ввода кода доступа. Активная сессия существует. |
| Результат успеха | Гость подключён к комнате презентации, видит текущий слайд и может отвечать на вопросы. |
| Триггер | Гость вводит код доступа и подтверждает, или переходит по прямой ссылке сессии. |
| Основной сценарий | 1. Гость переходит по ссылке или вводит код доступа на сайте.<br>2. Система проверяет валидность кода и наличие активной сессии.<br>3. Система включает (include) вариант использования «Ввод имени» (UC-7).<br>4. После получения имени система подключает зрителя к комнате презентации.<br>5. Система отображает зрителю текущий слайд.<br>6. Гость может отвечать на вопросы (опросы) по мере поступления. |
| Расширения | 2a. Код недействителен → сообщение «Сессия не найдена или истекла».<br>3a. Гость отказывается вводить имя → подключение не происходит. |

### UC-7. Ввод имени (включаемый)

| Элемент | Описание |
|---------|----------|
| Наименование | Ввод имени |
| Участники | Гость → цель: идентифицировать себя в сессии.<br>Система → цель: сохранить временный идентификатор гостя в сессии. |
| Предусловие | Гость прошёл проверку кода сессии (UC-6, шаг 2). |
| Результат успеха | В сессии сохранено имя гостя; гость допущен к просмотру и ответам. |
| Триггер | Система отображает диалог «Введите ваше имя» после успешной проверки кода. |
| Основной сценарий | 1. Система показывает поле ввода имени.<br>2. Гость вводит имя и подтверждает.<br>3. Система проверяет, что имя не пустое.<br>4. Система связывает имя с сокетом гостя и добавляет в комнату сессии. |
| Расширения | 3a. Имя пустое или состоит только из пробелов → сообщение «Имя не может быть пустым», возврат к шагу 1. |
