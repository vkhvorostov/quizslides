# Главный экран

---

## 1. Назначение и роль главного экрана

Главный экран — центральная точка входа в приложение. Он доступен по корневому URL и выполняет следующие функции:

- **Подключение к сессии** — любой пользователь (в том числе гость) может ввести 6-значный код и присоединиться к живой презентации.
- **Создание презентации** — после авторизации на главном экране появляется блок **«Вы организатор?»** с кнопкой **«Создать презентацию»**. По нажатию открывается всплывающее окно, в котором достаточно ввести название — и новый проект готов к работе.
- **Витрина возможностей платформы** — блок с карточками-превью функций: добавление контента на слайд, создание презентации, редактирование, AI-генерация слайдов (скоро).
- **Навигация** — верхняя панель с логотипом, поиском, профилем; ссылки на вход / регистрацию для гостей или приветствие + выход для авторизованных.

---

## 2. Структура страницы

### 2.1. Общий каркас

Адаптивная страница на Bootstrap 5 корректно работает на любых устройствах. Экран делится на три смысловых уровня сверху вниз:

| # | Область | Что видит пользователь |
|---|---------|----------------------|
| 1 | **Навигация** | Логотип **QuizSlides**, ссылки **Вход / Регистрация** (гость) или **Привет, username! / Выход** (авторизованный) |
| 2 | **Центральная зона** | Поле ввода кода сессии + кнопка **«Присоединиться»**; ниже — блок **«Вы организатор?»** с кнопкой **«Создать презентацию»** (только после входа) |
| 3 | **Витрина возможностей** | Горизонтальный ряд карточек 240×240 с превью функций платформы |

При нажатии **«Создать презентацию»** поверх страницы появляется **всплывающее окно** с формой ввода названия и кнопкой **«Создать»**.

### 2.2. Блоки главного экрана

| Блок | Назначение |
|------|------------|
| **Форма ввода кода** | Поле «ВВЕДИТЕ КОД» + кнопка «Присоединиться». Отправляет GET-запрос для подключения к сессии. |
| **Кнопка «Создать презентацию»** | Видна только авторизованным (`{% if user.is_authenticated %}`). Открывает попап с формой. |
| **Попап создания** | Форма с полем «Название презентации» + кнопка «Создать». Отправка через JS (`createPresentation(event)`), CSRF-токен. При успехе — сообщение «Презентация успешно создана!». |
| **Витрина «Возможности платформы»** | Карточки 240×240 с превью-картинками из `/static/drawable/`: `create_content_on_slide.png`, `create_new_presentation.png`, `edit_presentation.png`, `ai_slide_generator.png`. |

## 3. Сценарии взаимодействия

### Сценарий 1. Гость подключается к сессии

1. Пользователь (без авторизации) заходит на главную страницу.
2. Вводит **код сессии** в поле «ВВЕДИТЕ КОД».
3. Нажимает **«Присоединиться»**.
4. Система проверяет код и подключает участника к живой презентации.

### Сценарий 2. Организатор создаёт презентацию

1. Авторизованный пользователь видит на главной блок **«Вы организатор?»**.
2. Нажимает **«Создать презентацию»**.
3. Открывается попап с полем **«Название презентации»**.
4. Вводит название, нажимает **«Создать»**.
5. JS-функция `createPresentation(event)` отправляет данные на сервер (с CSRF-токеном).
6. При успехе — появляется сообщение **«Презентация успешно создана!»**.

### Сценарий 3. Первый визит — регистрация и вход

1. Новый пользователь открывает главную страницу — перед ним форма ввода кода сессии и ссылки **«Вход»** / **«Регистрация»** в навигации.
2. Переходит на `/accounts/signup/`, заполняет форму.
3. После регистрации — перенаправление на `/accounts/login/`, вход.
4. Возвращается на главную — теперь видна кнопка **«Создать презентацию»**.

### Сценарий 4. Просмотр возможностей

1. Любой пользователь прокручивает главную вниз.
2. Видит блок **«Возможности платформы»** — карточки с картинками: добавление контента, создание, редактирование, AI-генерация (скоро).

## 4. Код, отвечающий за главный экран

### 4.1. Форма подключения к сессии (фрагмент шаблона)

```html
<form method="GET" class="d-flex flex-column flex-md-row gap-3">
    <input type="text" name="code"
           class="form-control form-control-lg text-center border-0 bg-light shadow-none"
           placeholder="ВВЕДИТЕ КОД" style="width: 200px;" required>
    <button type="submit"
            class="btn btn-success btn-lg fw-semibold px-4 py-3">Присоединиться</button>
</form>
```

### 4.2. Блок организатора (только для авторизованных)

```html
{% if user.is_authenticated %}
<div class="d-flex flex-column flex-md-row justify-content-center
            align-items-center gap-3 mt-4">
    <span class="fs-4 fw-semibold text-dark">Вы организатор?</span>
    <button type="button" onclick="openPopup()" id="openPopupBtn"
            class="btn btn-success btn-lg fw-semibold px-4 py-3">
        Создать презентацию
    </button>
</div>
{% endif %}
```

### 4.3. Витрина возможностей

```html
<h2 class="display-6 fw-bold text-dark mb-5">Возможности платформы</h2>
<div class="d-flex flex-wrap justify-content-center gap-4">

    <div class="text-center" style="width: 240px;">
        <div class="rounded-4 shadow-sm overflow-hidden mb-2" style="height: 240px;">
            <img src="/static/drawable/create_new_presentation.png"
                 alt="Создание презентации" ...>
        </div>
        <p class="fw-semibold text-secondary">Создание презентации</p>
    </div>

    <!-- аналогично: edit_presentation, create_content_on_slide,
         ai_slide_generator -->
</div>
```

Статические картинки лежат в `/static/drawable/`.

### 4.4. Попап создания презентации

```html
<h5 class="text-center mb-3">Создание презентации</h5>
<form id="createPresentationForm" onsubmit="createPresentation(event)">
    {% csrf_token %}
    <div class="mb-3">
        <input type="text" name="title" id="presentationTitle"
               class="form-control" placeholder="Название презентации" required>
        <div class="invalid-feedback" id="titleError" style="display:none;"></div>
    </div>
    <div class="d-grid">
        <button type="submit" class="btn btn-success">Создать</button>
    </div>
</form>
<div id="successMessage" style="display:none;" class="alert alert-success mt-3">
    Презентация успешно создана!
</div>
```
