# QuizSlides
Веб-конструктор интерактивных презентаций с онлайн-конференциями

Дата начала разработки: 2026г.

Основные возможности:
1. Создание и редактирование презентаций
2.  Проведение интерактива с аудиторией (опрос, облако слов)

Поддерживаемые языки: русский, английский

## Быстрый старт
### 1. Создание и активация виртуального окружения
```bash
python -m venv venv
```
###### Для Windows:
```bash
venv\Scripts\activate
```
###### Для macOS/Linux:
```bash
source venv/bin/activate
```
### Установка зависимостей
```bash
pip install -r requirements.txt
```

### 2. Настройка переменных окружения
В репозитории есть пример `.env_example`, который нужно скопировать и заполнить:

```bash
cp .env_example .env
```

После этого откройте `.env` и укажите свои значения.  

| Переменная | Назначение |
|-------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **DB_NAME** | Имя базы данных PostgreSQL, которую будет использовать Django. |
| **DB_USER** | Имя пользователя PostgreSQL. |
| **DB_PASSWORD** | Пароль пользователя PostgreSQL. |
| **DB_HOST** | Адрес сервера БД. В случае локальной разработки — `localhost`. |
| **DB_PORT** | Порт подключения к БД (по умолчанию `5432`). |



## 3. Запуск PostgreSQL

Для работы приложения **QuizSlides** используется база данных **PostgreSQL**.

В проекте уже есть пример файла `docker-compose.yml`. 
Используйте готовый или измените под свои нужды, а дальше запускайте:

```bash
docker compose up -d
```

`-d` значит `detach`, то есть контейнер продолжит работать в фоне. Чтобы его остановить, надо будет ввести

```bash
docker compose down
```

Если проект запускается на чистой базе данных, примени миграции:
```bash
python manage.py migrate
```
### 4. Создание администратора
Для входа в админку нужен суперпользователь. Создать его можно командой:
```bash
python manage.py createsuperuser
```
После этого Django попросит указать логин, email и пароль.

 [Ссылка на вход в админку](http://127.0.0.1:8000/admin/)

### 5. Запуск проекта
После заполнения `.env` и настройки базы данных можно запустить сервер разработки:

```bash
python manage.py runserver
```

Проект доступен по адресу [http://localhost:8000](http://localhost:8000) или [http://127.0.0.1:8000](http://127.0.0.1:8000)
