# QuizSlides - Платформа для создания и прохождения квизов

Веб-приложение на Django для создания интерактивных квизов (опросов) с возможностью регистрации пользователей, авторизации и управления контентом.

## Технологии

- **Backend**: Django 4.2.29
- **Frontend**: HTML5, CSS3, Bootstrap 5
- **База данных**: SQLite3 (для разработки)
- **Аутентификация**: Django Auth System
- **Статические файлы**: Django Staticfiles
- **Язык**: Python 3.8+

##  Требования к системе

- Python 3.8 или выше
- pip (менеджер пакетов Python)
- Git (для клонирования репозитория)

##  Установка и запуск

### 1. Клонирование репозитория

```bash
git clone <URL-репозитория>
cd Django_glav
```

### 2. Создание виртуального окружения

Рекомендуется использовать виртуальное окружение для изоляции зависимостей:

```bash
# Для Windows
python -m venv venv
venv\Scripts\activate

# Для Linux/Mac
python3 -m venv venv
source venv/bin/activate
```

### 3. Установка зависимостей

Установите все необходимые пакеты из файла requirements.txt:

```bash
pip install -r requirements.txt
```

### 4. Настройка базы данных

Примените миграции для создания структуры базы данных:

```bash
python manage.py migrate
```

При необходимости создайте суперпользователя для доступа к админ-панели:

```bash
python manage.py createsuperuser
```

### 5. Запуск сервера разработки

Запустите сервер разработки Django:

```bash
python manage.py runserver
```
### 6. Доступ к приложению

После запуска сервера откройте браузер и перейдите по одному из адресов:

- Главная страница: `http://127.0.0.1:8000/`
- Страница входа: `http://127.0.0.1:8000/accounts/login/`
- Страница регистрации: `http://127.0.0.1:8000/accounts/signup/`
- Админ-панель: `http://127.0.0.1:8000/admin/` (требует суперпользователя)

##    Решение проблем

### 1. Ошибка "ModuleNotFoundError: No module named 'Django'"
Убедитесь, что виртуальное окружение активировано и зависимости установлены:
```bash
pip install -r requirements.txt
```

### 2. Ошибка миграций
Если возникают проблемы с миграциями, попробуйте:
```bash
python manage.py makemigrations
python manage.py migrate --run-syncdb
```

### 3. Порт уже занят
Если порт 8000 занят, используйте другой порт:
```bash
python manage.py runserver 8080
```

### 4. Проблемы со статическими файлами
В режиме разработки Django обслуживает статические файлы автоматически. Если файлы не отображаются:
- Убедитесь, что `DEBUG = True` в `quizslides/settings.py`
- Проверьте настройки `STATICFILES_DIRS` и `STATIC_URL`

### 5. Доступ с других устройств
Для доступа с других устройств в локальной сети:
```bash
python manage.py runserver 0.0.0.0:8000
```
Затем откройте `http://<ваш-ip-адрес>:8000/` на другом устройстве.

##  Примечания

- Проект использует SQLite для простоты разработки. Для продакшена рекомендуется использовать PostgreSQL или MySQL.
- Все настройки находятся в `quizslides/settings.py`. Для продакшена необходимо:
  - Установить `DEBUG = False`
  - Настроить `ALLOWED_HOSTS`
  - Использовать безопасный `SECRET_KEY`
  - Настроить сервер для статических файлов (Nginx/Apache)
